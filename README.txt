--- FOLDERS AND INFO ---

documents: Contains the documents for Group 3 in Project 2

The documents folder contains the Game Design Document, the High Concept Document, and the Team Contract in the form of downloadable PDFs

build: Contains the Gameplay Prototype for Group 3 in Project 2

The build folder contains a fully functional build of our game prototype Saddle Skedaddle, with folders and a crash-handling file that helps the game run. 

In the game Saddle Skedaddle, the "Guard" is the enemy Cowboy, and the McGuffin being the player's trusty ol' Saddle that's locked away in the lazy sheriff's shed. There are other obstacles like barrels, a horse carriage, tumbleweeds, and even a dust devil if you dilly dally. As well, there is the nasty Sheriff who chases you along with the Cowboy when you try getting his saddle. 

- In order to run the game, double-click the application called Saddle Skedaddle
- to exit, press Quit in the Main Menu, Pause, Win, and Game Over Screen

Controls for Keyboard and Mouse/Controller
- WASD/Joystick: movement
- Shift/Left Trigger: sprint
- Esc/Start: Pause
- Mouse/Left Joystick: camera control
- Enter/A: cutscene skip
- Y/Y: YEEHAW

Aside from crash-handling and the game itself, these folders will be present as well.

D3D12: Allows games to use modern graphics features without requiring an update for the client
MonoBleedingEdge: Folder that contains run-time for using C# code in games
Saddle Skedaddle_Data: Data for the game

--- INTENDED ROUTE ---

The intended route if you're struggling is to run either left or right to dodge the cowboy's duel, then enter the stable, and run towards the sheriff's shed, past the jail. The sheriff and the saddle will both be there. Pick up the saddle while using your sprint to dodge the sheriff, and then skedaddle to the stables!
